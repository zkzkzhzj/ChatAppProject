### 상세

#### F-1: 모바일 터치 이동

현재 키보드(WASD/방향키)로만 이동 가능. 모바일에서는 입력 수단이 없어 캐릭터가 움직이지 않음.

- 방안 A: 터치한 위치로 pathfinding 이동
- 방안 B: 가상 조이스틱 UI

#### F-2: 떠난 유저의 typing 말풍선 고아 잔존

다른 유저 A 가 채팅 입력 중(머리 위 typing dots 표시)일 때 브라우저를 닫는 등 비정상 종료하면, 다른 클라이언트에서 A 의 캐릭터는 정상적으로 사라지지만 **typing 말풍선만 마지막 위치에 그대로 남아 떠 있는** 현상.

원인: `VillageScene.removeOtherPlayer` / `sweepStalePlayers` 가 `entry.container` 만 destroy 하고 `entry.bubble` 의 destroy 를 누락. Phaser scene 에 별도 등록된 bubble 객체가 부모 entry 삭제 후에도 남아 고아 상태가 됨.

수정 (커밋 예정): 두 메서드 모두 `entry.bubble?.destroy()` 를 캐릭터 destroy 와 함께 호출하도록 보강.

#### F-3: 맥북 IME 반복 입력

macOS + 한글 IME 조합에서 채팅 입력 시 마지막 글자/단어가 중복 입력되는 문제. `compositionend` 이벤트 핸들링 누락 가능성.

#### F-4: 음성 채팅

WebRTC 기반 음성 채팅. Phase 로드맵에 이미 포함되어 있음. 마을 내 근접 유저 간 음성 대화.

#### F-5: 회원가입 고도화

현재 이메일 + 비밀번호만으로 가입. 유저 이름(닉네임)을 설정하고, 채팅/마을에서 이름으로 표시되어야 함.

## 기술 부채 (2026-04-17 기준)

### MVP 테스트 목적으로 의도적으로 단순화한 것들

- 로컬 Docker 빌드 (CI/CD 미구축) — 빠른 반복 우선
- 단일 서버 (로드밸런싱 없음) — 소수 테스트 유저 대상
- nginx 호스트 설치 (Docker 외부) — 빠른 설정 우선
- Flexible SSL (완전 암호화 아님) — 무료 + 무설정 우선
