# Brainstorming

## 목적

모호한 요청을 구현 가능한 목표, 비목표, 성공 조건으로 정리한다.

## 사용 조건

- 사용자의 요구가 한 문장이고 범위가 넓다.
- 설계 선택지가 여러 개이며 비용 차이가 크다.
- 기능 이름은 있지만 완료 조건이 불명확하다.
- 바로 구현하면 되돌릴 가능성이 높다.

## 절차

1. 사용자가 원하는 결과를 한 문장으로 재진술한다.
2. 목표와 비목표를 분리한다.
3. 사용자 흐름, 데이터 흐름, 실패 조건을 질문한다.
4. 가능한 접근 2-3개와 트레이드오프를 제시한다.
5. 선택된 방향을 짧은 설계 요약으로 고정한다.
6. 구현 범위가 크면 `writing-plan.md` 또는 `track-start.md`로 넘긴다.

## 전이 규칙

브레인스토밍은 대화 종료 지점이 아니다.

- 목표, 비목표, 성공 조건이 합의되면 메인 Codex는 즉시 다음 단계를 선택한다.
- 새 기능, 새 API, DB 변경, UI 흐름이 있으면 `writing-plan.md`로 전이한다.
- 도메인/API/보안/테스트처럼 독립 검토가 필요하면 `parallel-agent-dispatch.md`를 먼저 실행한다.
- 장기 작업 또는 병행 작업이면 `track-start.md`로 전이한다.
- 사용자가 "여기서 멈춰", "검토만", "문서만"이라고 말한 경우에만 종료한다.
- 다음 단계에 필요한 결정이 빠졌다면 열린 질문 1-3개만 묻고 이어서 진행한다.
