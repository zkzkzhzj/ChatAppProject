# Spec New

## 목적

새 기능이나 의미 있는 변경을 시작하기 전에 `docs/specs/features/{feature}.md`를 만든다.

## 절차

1. 관련 planning, architecture, convention 문서를 확인한다.
2. `docs/specs/features/_template.md`를 기준으로 spec을 만든다.
3. outcomes, scope, constraints, decisions, tasks, verification을 채운다.
4. decisions에는 왜, 대안, 빈틈, 재검토 조건을 가능한 한 먼저 적는다.
5. 불명확한 비즈니스 결정은 Main Orchestrator가 사용자에게 묻는다.
