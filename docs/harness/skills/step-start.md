# Step Start

## 목적

한 트랙 안에서 1 step = 1 PR 원칙으로 구현 단계를 시작한다.

## 절차

1. 현재 track 문서의 로드맵과 step 상태를 확인한다.
2. 이번 step의 in/out scope를 확정한다.
3. 필요한 Agent를 고른다.
4. 위험도에 따라 Technical Strategy, Security, Concurrency Critic을 붙인다.
5. 성공 기준을 검증 가능한 문장으로 적는다.
6. 구현 후 테스트와 리뷰 게이트를 수행한다.
