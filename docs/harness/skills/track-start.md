# Track Start

## 목적

병행 작업 충돌을 피하기 위해 새 작업 트랙을 시작한다.

## 절차

1. 대응 spec이 있는지 확인한다.
2. `docs/handover/INDEX.md`에서 활성 트랙을 확인한다.
3. `docs/handover/track-{id}.md`를 만든다.
4. 충돌 위험 파일을 명시한다.
5. 필요하면 `docs/learning/RESERVED.md`에 학습 노트 번호를 예약한다.
6. 브랜치 이름은 `docs/conventions/git.md` 규칙을 따른다.
