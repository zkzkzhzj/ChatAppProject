## ERD 물리 설계 확정

**결정 사항:**
- CHARACTER ↔ USER: **1:1** 확정 (`CHARACTER.user_id UNIQUE` 제약)
- SPACE ↔ USER: **1:N** 유지 (`is_default`로 기본 공간 구분)
- 아이템 장착/배치: `USER_ITEM_INVENTORY` 직접 참조 없이 `ITEM_DEFINITION`만 참조
  - 소유 검증은 애플리케이션 레이어에서 수행
  - 아이템은 소모재가 아닌 소유재 — 장착/배치해도 인벤토리에서 사라지지 않음
  - `SPACE_PLACEMENT`는 동일 아이템 무제한 배치 허용 (수량 컬럼 없음)
- `POINT_WALLET` 분리 유지: transaction 합산 방식 대신 스냅샷 + 낙관적 락 방식 채택
  - 이유: 동시 차감 시 레이스 컨디션 방지, O(1) 잔액 조회
