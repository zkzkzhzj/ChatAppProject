# Project Overview — 마음의 고향

---

## 1. 서비스 비전

**시작점:** 대화가 그리운 사람, 누군가의 온기가 필요한 사람이 편하게 찾아와 고향에 온 듯한 느낌을 받을 수 있는 곳.

무거운 고민이 있어서가 아니라, 그냥 오늘 좀 외롭거나, 사람이 그립거나, 따뜻한 대화가 필요할 때 — 그때 들어올 수 있는 곳이다.

그러나 단순한 채팅 서비스로는 유저가 이 서비스를 써야 할 이유가 없다. 카카오톡 익명채팅, 블라인드가 이미 존재한다.

여기서 "고향"이라는 키워드가 방향을 잡아줬다. 고향은 장소다. 장소에는 마을이 있고, 마을에는 이웃이 있고, 내 집이 있다. 이 서비스 자체가 하나의 **마을**이 되는 것이다.

유저는 마을에 들어와 돌아다니고, 이웃을 만나 대화하고, 자기 공간을 꾸미며 소속감을 느낀다. 채팅은 "기능"이 아니라 마을에서 살아가면서 자연스럽게 일어나는 "행위"가 된다.

---

## 2. 핵심 가치

**"대화가 그리운 사람을 위한 안식처"**

이 한 문장이 모든 기능과 설계 결정의 기준이다.

- 들어오는 순간 "여기는 편한 곳이다"를 느껴야 한다.
- 가벼운 일상 수다부터 마음속 이야기까지, 감정의 무게와 관계없이 누구나 환영받는다.
- 대화의 톤은 따뜻하고, 비난은 차단되어야 한다.
- 꾸미기는 단순 재미가 아니라, "내 안식처를 만드는 행위"다.

---

## 3. 핵심 기능

### 3.1 공간 / 장소 (Space)

**역할:** 유저 유입의 핵심 전략.

서비스에 진입하면 하나의 마을이 펼쳐진다. 마을은 인터랙티브 2D 공간으로, 배경과 건물, NPC가 배치되어 있다. 유저는 자신만의 공간(집)을 가지며, 이 공간을 꾸미는 것이 서비스의 핵심 재미 요소다.

- 귀여운 비주얼로 타겟 유저의 감성을 자극한다.
- 공간이 있기 때문에 "돌아올 이유"가 생긴다.

### 3.2 캐릭터 이동 (Character)

**역할:** 마을에 실제로 살고 있다는 몰입감 제공.

유저는 캐릭터를 조작하여 마을을 돌아다닌다. 단순히 세워두는 것이 아니라, 마을을 탐색하고 다른 유저나 NPC와 상호작용하면서 "하나의 마을에서 살아가는 느낌"을 받을 수 있도록 한다.

### 3.3 채팅 (Chat)

**역할:** 서비스의 핵심 기능. 유저 간 소통.

마을에서 다른 유저 또는 NPC에 근접하면 대화가 가능하다. 텍스트 채팅을 기본으로 제공한다.

- 텍스트 채팅: 실시간 WebSocket 기반
- 음성 / 화면 공유: WebRTC 기반 — **추후 추가 예정**
- 메시지 필터링: 악성 채팅, 비난 감지 및 차단

### 3.4 포인트 / 아이템 (Point & Item)

**역할:** 공간 꾸미기의 경제 시스템.

유저는 포인트를 획득하여 아이템을 구매한다. 아이템은 캐릭터 꾸미기 또는 공간 디자인에 사용된다.

- 포인트 획득: 광고 시청 (리워드 광고)
- 포인트 소비: 캐릭터 외형 아이템, 공간 배경/가구 아이템
- 포인트 경제의 동시성과 정합성이 기술적으로 중요한 영역이다.

**확장 요소 (현재 범위 아님):**
- 상대방에게 좋아요/응원 받기로 포인트 획득
- 일일 미션 클리어 (출석, 대화 N회 등)
- 마을 내 미니 미션 수행

### 3.5 AI NPC (마을 주민)

**역할:** 초기 유저 부족 문제 해결 + 정서적 보조.

마을에 항상 존재하는 NPC다. 유저가 적은 초기에 "빈 마을" 경험을 방지하고, 대화가 그리운 유저에게 따뜻한 대화 상대가 되어준다.

- MVP: 마을 내 고정 위치에 "마을 지기" NPC 1개. 정해진 대화 패턴 + 간단한 AI 응답.
- 확장: NPC 수 증가, 간단한 이동 패턴 추가.
- 유저가 충분히 늘어나면 AI 의존도를 점진적으로 줄인다.
- 위험 신호 감지 시 전문 상담 리소스 안내 (윤리적 필수 요소).

**확장 고려 (현재 범위 아님):**
- 로컬 LLM 기반의 더 자연스러운 대화 상호작용
- NPC별 고유 페르소나 (마을 지기, 카페 주인, 산책하는 이웃 등)
- 유저의 대화 맥락을 기억하여 지속적인 관계 형성

## 4. 서비스 흐름

```
1. 유저 진입
   → 로그인 없이 바로 마을에 입장할 수 있다.
   → 기본 게스트 캐릭터가 자동 부여된다.

2. 마을 탐색 (비로그인)
   → 게스트 캐릭터로 마을을 돌아다닌다.
   → NPC와 다른 유저가 보인다. 마을의 분위기를 체험한다.

3. 상호작용 시도 → 로그인 유도
   → 대화, 친구 추가, 공간 꾸미기 등을 시도하면 로그인을 요청한다.
   → 소셜 로그인 우선으로 간소화.

4. 자기 캐릭터 및 공간 할당
   → 로그인 후 기본 캐릭터와 기본 배경이 할당된다.
   → 자기 공간(집)이 생성된다.

5. 마을 생활
   → NPC 또는 다른 유저에 근접하면 대화 가능.
   → 텍스트 채팅. (음성, 화면 공유는 추후 추가 예정)

6. 공간 꾸미기
   → 자기 공간(집)으로 이동하여 아이템을 배치.
   → 다른 유저가 내 공간을 방문할 수 있다.

7. 포인트 획득 및 아이템 구매
   → 광고 시청으로 포인트 획득.
   → 상점에서 아이템 구매 → 캐릭터 또는 공간에 적용.
```

## 5. 수익 모델

### 리워드 광고

유저가 자발적으로 광고를 시청하고 포인트를 획득하는 구조.

- 유저 경험을 해치지 않는 비침투적 광고.
- 강제 광고 없음. 유저의 선택에 의해서만 광고가 재생된다.
- 광고 시청 → 포인트 지급의 데이터 정합성이 중요하다. (광고는 봤는데 포인트가 안 들어오는 상황 방지)

### 향후 확장 가능성 (현재 범위 아님)
- 프리미엄 아이템 (유료 구매)
- 구독 모델 (광고 제거 + 추가 혜택)

## 6. 경쟁 서비스와의 차별점

| 서비스 | 특징 | 마음의 고향과의 차이 |
|--------|------|---------------------|
| 카카오톡 익명채팅 | 텍스트 채팅 + 보이스톡 제공 | 공간이 없음. 감성적 경험 없음. 범용 메신저의 부가 기능 수준. |
| 블라인드 | 직장인 익명 커뮤니티 | 특정 타겟(직장인) 한정. 공간 개념 없음. |
| ZEP | 범용 2D 가상 공간 | "누구를 위한 것인지"가 없음. 마을이라는 세계관과 정서적 목적이 없음. |
| 게더타운 | 업무용 가상 오피스 | 업무 중심. 감성적 안식처 개념 없음. |

**마음의 고향의 차별점:** 기술이 아니라 경험의 목적이 다르다. "대화가 그리운 사람을 위한 따뜻한 마을"이라는 명확한 정서적 목적이 있고, 모든 기능이 이 목적을 위해 설계된다. 또한 웹 기반이라 앱 설치 없이 브라우저에서 바로 접근할 수 있어, 온기가 필요한 순간에 찾아오는 유저의 진입 장벽을 최소화한다.

---

## 7. 플랫폼 전략

- **웹 (데스크탑 우선)**: 별도 앱 설치 없이 브라우저에서 바로 접근 가능.
- **모바일 웹 대응**: 채팅 등 기본 기능은 모바일 브라우저에서도 동작하되, 인터랙티브 공간은 데스크탑 최적화.
- **알림**: FCM 기반 Web Push로 재방문 유도.

---

## 8. 미해결 과제

이 문서에서 아직 확정되지 않은 항목들. 개발 과정에서 점진적으로 결정한다.

- 마을의 구체적인 맵 구성 (크기, 구역 분할, 건물 배치)
- 동시 접속자 증가 시 채널링(자동 배정) 도입 시점과 채널당 적정 인원
- AI NPC의 대화 톤과 페르소나 상세 정의
- 아이템 카테고리 및 가격 체계
- 지역구/소속 시스템의 도입 여부
- 광고 파트너 및 SDK 선정
