# Project Overview - 마음의 고향

---

## 1. 서비스 비전

**마음의 고향**은 대화가 그리운 사람이 조용히 들어와 머물고, 말하기 어려운 마음을 안전하게 남기고, 누군가의 편지를 통해 위로를 받는 장소 기반 의사소통 서비스다.

마을과 도서관은 제품의 첫 인상이다. 사용자는 3D 공간에 들어와 움직이고, 다른 사용자의 존재를 보고, 도서관으로 이동한다. 그러나 장기적으로 저장되고 제품의 중심축이 되는 데이터는 런타임 장면 자체가 아니라 고백, 편지, 감사 답장, 반응/신고, 그리고 비공개 사서 RAG가 참조하는 사적 고백/도서관 맥락이다.

---

## 2. 핵심 가치

**"대화가 그리운 사람을 위한 안식처"**

- 들어오는 순간 "여기는 안전한 곳"이라는 감각을 준다.
- 가벼운 일상 대화부터 무거운 마음까지 비난 없이 담을 수 있게 한다.
- 일반 채팅의 순간성을 고백/편지의 사적 기록과 분리한다.
- 사서 RAG는 사용자의 고백과 도서관 맥락 안에서만 작동하며, 일반 채팅을 장기 기억으로 삼지 않는다.

---

## 3. 제품 축

### 3.1 3D 마을 / 도서관

**역할:** 서비스 진입 경험과 장소 감각 제공.

마을과 도서관은 Three.js 기반 런타임 장면이다. 사용자는 기본 아바타로 이동하고, WebSocket을 통해 위치가 공유되며, 장면 전환으로 도서관에 들어간다.

- 위치와 아바타 상태는 런타임 상태다.
- 사용자별 장면이나 아바타 설정을 DB에 저장하지 않는다.
- 도서관은 고백/편지/사서 RAG로 이어지는 정서적 입구다.

### 3.2 고백

**역할:** 마음을 남기는 핵심 저장 표면.

사용자는 제목과 본문을 가진 고백을 남긴다. 고백은 책장, 상태, 위험 수준 등 안전한 운영에 필요한 메타데이터를 가진다.

- 고백 기록은 장기 제품 축이다.
- 마을 대시보드는 오늘 남겨진 고백 수를 보여줄 수 있다.
- 위험 신호와 신고 흐름은 Safety와 연결된다.

### 3.3 편지와 감사 답장

**역할:** 사용자가 서로에게 위로를 건네는 비동기 소통.

고백에 대해 편지를 보내고, 고백 작성자는 감사 답장을 남길 수 있다. 이 흐름은 공개 채팅과 달리 사적인 위로의 맥락을 보존한다.

- 편지는 고백에 속한다.
- 감사 답장은 편지에 한 번 연결된다.
- 반응과 신고는 중복 방지 제약으로 관리한다.

### 3.4 비공개 사서 RAG

**역할:** 도서관 경험을 고백/편지 데이터와 연결하는 사적 도움말 축.

사서 RAG는 고백과 도서관의 사적 맥락을 경계로 한다. 구체적인 저장 테이블은 구현된 스키마에 맞춰 별도 설계하며, 이 문서는 아직 구현되지 않은 테이블을 현재 ERD로 선언하지 않는다.

- 일반 채팅 메시지는 사서 RAG의 장기 기억이 아니다.
- 고백/편지 경계의 민감 데이터는 Confession/Library 책임으로 다룬다.
- 안전 정책과 삭제/접근 권한을 우선 설계한다.

### 3.5 공개 채팅

**역할:** 마을 안에서 일어나는 순간적 대화.

공개 채팅은 실시간 의사소통 기능이다. 메시지는 채팅 도메인에서 다루며, 사서 RAG의 개인 기억으로 승격하지 않는다.

---

## 4. 사용자 흐름

```text
1. 사용자가 마을에 진입한다.
   -> 기본 아바타가 런타임에 표시된다.
   -> 오늘 방문이 한 번 집계된다.

2. 사용자가 마을과 도서관을 이동한다.
   -> 위치는 WebSocket으로 공유된다.
   -> 다른 사용자는 RemotePlayer로 보인다.

3. 사용자가 도서관에서 고백을 남긴다.
   -> confession_record에 저장된다.
   -> 오늘 대시보드 집계에 반영된다.

4. 다른 사용자가 편지를 보낸다.
   -> confession_letter에 저장된다.
   -> 작성자는 편지를 읽고 감사 답장을 남길 수 있다.

5. 사서 RAG가 도서관 경험을 보조한다.
   -> 고백/도서관 사적 데이터 경계 안에서만 맥락을 구성한다.
```

---

## 5. 수익 모델

이번 트랙에서는 수익 모델을 설계하지 않는다. 현재 제품 문서는 수익화 기능을 대기 중인 제품 계획으로 두지 않고, 고백/편지/사서 RAG 중심의 저장 표면을 먼저 정렬한다.

---

## 6. 경쟁 서비스와 차별점

| 서비스 | 특징 | 마음의 고향과의 차이 |
|--------|------|---------------------|
| 익명 채팅 서비스 | 텍스트 중심 실시간 대화 | 장소 경험과 고백/편지 흐름이 약하다. |
| 직장 커뮤니티 | 특정 조직 기반 익명 대화 | 정서적 안식처보다 정보 교환에 가깝다. |
| 메타버스 공간 서비스 | 범용 가상 공간 | 고백과 도서관이라는 사적 기록 축이 없다. |
| 업무형 가상 오피스 | 협업과 회의 중심 | 대화가 그리운 사용자의 정서적 회복 목적과 다르다. |

마음의 고향의 차별점은 3D 공간 자체가 아니라, 공간을 입구로 삼아 고백과 편지, 비공개 사서 RAG로 이어지는 정서적 제품 축에 있다.

---

## 7. 현재 미해결 과제

- 사서 RAG의 접근 권한, 삭제 정책, 안전 정책 확정
- 고백/편지 UX와 신고/운영자 처리 흐름 정교화
- 공개 채팅과 사적 고백 데이터 경계에 대한 테스트와 문서 보강
- 마을/도서관 런타임 성능과 모바일 대응 범위 결정
