# Target User — 마음의 고향

---

## 1. 타겟 유저

### 1.1 핵심 감정

이 서비스의 타겟은 특정 나이대나 직업이 아니라, **특정 감정 상태**에 있는 사람이다.

- "오늘 좀 외롭다"
- "누군가랑 가볍게 이야기하고 싶다"
- "사람의 온기가 그립다"
- "혼자가 아니라는 느낌을 받고 싶다"

무거운 고민이 있어서가 아니다. 그냥 대화가 그리운 것. 이 감정은 10대도, 20대도, 30대도, 누구나 느낀다. 나이가 아니라 감정이 타겟이다.

### 1.2 페르소나

타겟을 가두지는 않되, 서비스를 설계할 때 떠올릴 대표 유저상을 정의한다.

**페르소나 A — 자취생 (20대 초중반)**

- 혼자 사는 원룸에서 저녁을 먹고 있다.
- SNS는 하지만, 피드를 보는 건 대화가 아니다.
- 가볍게 "오늘 뭐 먹었어?" 같은 이야기를 나눌 상대가 그립다.
- 귀여운 것을 좋아하고, 픽셀아트/도트 감성에 반응한다.

**페르소나 B — 번아웃 직장인 (20대 후반~30대)**

- 퇴근 후 지쳐서 누워있지만, 잠이 안 온다.
- 직장 동료와의 대화는 업무 이야기뿐이다.
- 이해관계 없이 편하게 이야기할 수 있는 공간이 필요하다.
- 조용히 읽고, 부담 없는 편지로 응답하는 낮은 압력의 소통을 원한다.

**페르소나 C — 새로운 환경에 적응 중인 사람**

- 이사, 전학, 이직 등으로 아는 사람이 없는 환경에 왔다.
- 새로운 관계를 만들기까지 시간이 걸린다.
- 그 사이의 외로움을 달래줄 곳이 필요하다.

### 1.3 공통 특성

- 대화 자체가 목적이다. 정보 교환이나 네트워킹이 아니다.
- 익명이거나 반익명(캐릭터 기반)인 환경을 선호한다.
- 무겁지 않은, 따뜻한 분위기를 원한다.
- 귀여운 비주얼에 끌린다.

---

## 2. 비주얼 전략

### 2.1 픽셀 / 도트 아트

서비스의 비주얼 방향은 **픽셀(도트) 아트**다.

**선택 이유:**

- AI 에셋 생성과의 궁합이 좋다. 픽셀아트는 스타일 일관성을 유지하기 가장 쉬운 아트 스타일이다.
- Phaser.js의 타일맵, 스프라이트 시스템이 픽셀아트에 최적화되어 있다.
- "따뜻한 레트로" 감성이 "고향"이라는 컨셉과 자연스럽게 어울린다.
- X(Twitter), 인디 게임 커뮤니티에서 픽셀아트 콘텐츠의 반응이 좋다. 유입 채널로 활용 가능하다.

### 2.2 비주얼 톤

- 따뜻한 색감 (파스텔, 따뜻한 조명)
- 귀여운 캐릭터 (둥글고 심플한 디자인)
- 편안한 마을 분위기 (나무, 작은 집, 가로등, 벤치)
- 공격적이거나 자극적인 요소 배제

---

## 3. 유입 전략

### 3.1 1차 유입 — X (Twitter)

픽셀아트 콘텐츠가 X에서 높은 반응을 얻는 점을 활용한다.

- 마을 스크린샷, 캐릭터 애니메이션 GIF 공유
- "오늘의 도서관 풍경", "오늘의 고백 문장" 같은 시리즈 콘텐츠
- 픽셀아트 커뮤니티, 인디 게임 커뮤니티와의 접점

### 3.2 2차 유입 — 입소문

서비스 경험 자체가 공유 콘텐츠가 되도록 설계한다.

- "오늘의 도서관 장면" 또는 공개 고백 카드 공유
- 인상 깊은 고백/편지 경험을 직접 공유하고 싶은 흐름

### 3.3 진입 장벽 최소화

- 웹 기반이라 앱 설치 불필요. URL 클릭만으로 진입.
- **회원가입 없이 마을 도서관과 공개 읽기 표면을 둘러볼 수 있다.** 고백 작성, 편지 보내기, 건의 등록 같은 쓰기 행동 시점에 로그인을 유도한다.
- 회원가입은 소셜 로그인 우선으로 간소화.
- 진입 직후 마을 도서관과 공개 서가가 바로 보인다. 튜토리얼이 길지 않다.

---

## 4. 리텐션 전략

유저가 다시 돌아오는 이유를 만든다.

- **고백 / 편지**: 누군가의 이야기를 읽고 편지를 남기며 낮은 압력의 연결감을 얻는다.
- **도서관 / 서가**: 다시 들어오면 새 고백과 답장을 확인할 수 있다.
- **개인별 사서**: 내가 남긴 고백과 받은 편지의 맥락을 바탕으로 돌아볼 질문을 던지고 응답을 받는다.
- **Web Push 알림**: "새 편지가 도착했어요", "감사 답장이 왔어요" 같은 재방문 유도.
