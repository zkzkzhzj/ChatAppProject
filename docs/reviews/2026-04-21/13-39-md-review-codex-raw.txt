 211: 클라이언트 STOMP CONNECT (Authorization: Bearer {token})
 212:   → StompAuthChannelInterceptor → JWT 파싱 → Principal 설정
 213: 클라이언트 → /app/chat/village (body만 전송)
 214:   → ChatMessageHandler → SendMessageUseCase.Command(userId, publicChatRoomId, body)
 215:   → getOrCreateParticipant() (첫 메시지 시 자동 참여, V4 UNIQUE 보호)
 216:   → Message(유저) → Cassandra 저장 (message + user_message dual-write)
 217:   → /topic/chat/village broadcast: MessageResponse(user) — 즉시 반환
 218:   → [비동기] NpcReplyService.replyAsync()
 219:     → 유저 메시지 임베딩 → pgvector 유사도 검색 → 맥락 주입
 220:     → LLM 응답 생성 → Cassandra 저장
 221:     → /topic/chat/village broadcast: MessageResponse(npc)
 222:   → [3회 누적 시] Outbox → Kafka → 대화 요약 → pgvector 저장
 223: 
 224: [REST 경로 — fallback]
 225: POST /api/v1/chat/messages {body: "..."}
 226:   → 동일 UseCase 실행
 227:   → SimpMessagingTemplate으로 /topic/chat/village broadcast (유저 메시지만)
 228:   → REST 응답: {userMessage} (NPC 응답은 비동기 WebSocket)
 229: ```

 504: 
 505: - **첫 배포는 반드시 `force_rebuild=true`**. 그래야 sha + latest 태그가 GHCR에 처음 생성됨
 506: - **`git reset --hard`가 deploy.sh에 포함**됨 → EC2에서 수동 디버깅 중이던 변경은 날아감 (의도된 동작, 결정적 상태 보장)
