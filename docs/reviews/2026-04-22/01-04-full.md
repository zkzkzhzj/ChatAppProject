### [P1/CRITICAL] Prometheus 엔드포인트 운영 기본값 공개 위험
**근거:** `deploy/docker-compose.yml:177` + `deploy/docker-compose.yml:182`

- `ACTUATOR_ENDPOINTS` 기본값 = `health,info,metrics,prometheus`
- `SECURITY_ENV_PUBLIC_PATHS` 기본값 = `/actuator/**` (SecurityConfig가 이 경로를 무인증 허용)
- 조합 결과: `.env`가 없거나 누락된 배포에서는 `/actuator/prometheus`가 **인증 없이 노출**됨
- Security Group 차단은 런타임 네트워크 방어일 뿐, compose 기본값의 "공개 노출" 자체를 막지 못함
- 주석("SG로 차단")이 디폴트 보안을 보장하지 않는다는 것이 핵심

### [P1/CRITICAL] Grafana 기본 비밀번호 허용
**근거:** `monitoring/docker-compose.yml:54`

- `GF_SECURITY_ADMIN_PASSWORD: ${GRAFANA_ADMIN_PASSWORD:-changeme}`
- Grafana는 `3001:3000`으로 **public bind**
- 모니터링 EC2에서 `.env` 생성이 누락되면 즉시 `admin/changeme`로 인터넷 노출
- 수정 제안: `${GRAFANA_ADMIN_PASSWORD:?GRAFANA_ADMIN_PASSWORD required}` — 설정 누락 시 compose 기동 자체를 실패시키기

### [P2/WARNING] Prometheus scrape target 고정 Private IP 하드코딩
**근거:** `monitoring/prometheus/prometheus.yml:20`

- `targets: ['172.31.45.140:8080']` — 운영 EC2의 Private IPv4가 yml에 박힘
- 운영 EC2 재생성/Auto Recovery 시 Private IP가 바뀌면 **모니터링만 조용히 끊김** (알람도 끊기므로 발견이 늦어짐)
- 대안: env 템플릿화, EIP/ENI 고정, 또는 README에 "Private IP 고정 전제" 명시
